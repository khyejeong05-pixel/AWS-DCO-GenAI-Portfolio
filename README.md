# 🤖 9. 생성형 AI 활용 원칙

생성형 AI는 다음과 같은 원칙 아래 활용했습니다.

- AI의 결과는 초안으로 활용
- 원본 로그와 비교하여 검증
- 확인된 사실과 추정을 구분
- Root Cause를 임의로 확정하지 않음
- Ticket Priority는 직접 판단
- 실제 AWS 정보 입력 금지
- 교육용 샘플 데이터만 활용

# 📖 11. 배운 점

이번 프로젝트를 통해 다음 내용을 학습했습니다.

- 생성형 AI를 활용하면 문서 작성과 코드 작성 시간을 줄일 수 있습니다.
- AI 결과는 반드시 검증 과정이 필요합니다.
- 로그 분석에서는 사실과 추정을 구분해야 합니다.
- 장애 복구와 Root Cause 확인은 서로 다른 개념입니다.
- Severity와 Priority의 차이를 이해했습니다.
- Incident Report는 다른 사람이 이해할 수 있도록 작성해야 합니다.
- Shift Handover는 다음 근무자가 업무를 이어갈 수 있도록 작성해야 합니다.
- Git과 GitHub를 활용한 버전 관리의 중요성을 이해했습니다.
- AI 활용 과정 자체도 중요한 기술 역량이라는 점을 배웠습니다.
