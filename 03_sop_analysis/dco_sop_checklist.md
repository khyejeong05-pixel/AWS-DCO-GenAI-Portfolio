---

## 1. 내가 읽은 교육용 샘플 SOP 제목

SAMPLE-DCO-SOP-04: 샘플 Link Down 이벤트 기록 및 전달 절차

---

## 2. SOP의 목적

내가 이해한 목적: 이 샘플 SOP는 교육용 가상 로그에서 Link Down(연결 끊김) 이벤트가 발생했을 때, 데이터센터 운영자로서 꼭 챙겨야 할 핵심 정보(시간, 장비명, 영향 범위, 전파 여부)를 식별하고 정리하는 문서화 연습을 목적으로 합니다.

---

## 3. SOP 절차 요약

1. 기본 정보 식별: 샘플 로그에서 이벤트가 발생한 정확한 시간과 샘플 장비명을 찾아 기록합니다.

2. 패턴 및 상태 확인: 해당 장비에서 동일한 이벤트가 반복해서 일어났는지 확인하고, 티켓 오픈(Ticket opened)이나 상위 전파(Ticket escalated) 같은 진행 상황 표현이 있는지 점검합니다.

3. 사실 기반 기록: 로그에 적힌 현상만 그대로 정리하며, 임의로 원인을 단정 짓지 않습니다.

4. 후속 조치 판단: 혼자 판단하기 모호한 내용은 '상위 전달(Escalation) 필요'로 분류합니다.

5. 초안 작성: 확인된 내용을 바탕으로 인수인계 메모(Handover Memo)나 장애 보고서(Incident Report)에 들어갈 수 있게 핵심만 요약합니다.

---

## 4. 주의사항

- 실제 장비에 접속하지 않습니다.
- 실제 네트워크 진단을 하지 않습니다.
- 실제 AWS 환경에 접속하지 않습니다.
- 실제 티켓 번호, 실제 장비 시리얼, 실제 IP를 사용하지 않습니다.
- AI 답변을 그대로 복사하지 않고 검토합니다.

## 5. 확인항목

□ 발생 시간을 확인했습니다.
□ 샘플 장비명을 확인했습니다.
□ 반복 여부를 확인했습니다.
□ 관련 Ticket 표현을 확인했습니다.
□ Escalation 필요 여부를 표시했습니다.
□ 실제 정보가 포함되지 않았는지 확인했습니다.
□ AI 답변에서 위험한 표현을 수정했습니다.

## 6. 내가 만든 SOP 체크리스트

[1단계: 샘플 로그 분석 및 기록]

[ ] 샘플 로그에서 Link Down 이벤트가 발생한 정확한 시간을 확인하고 기록했는가?

[ ] 이벤트가 발생한 샘플 장비명(SAMPLE, DEMO, EDU 접두어)을 확인하고 기록했는가?

[ ] 해당 샘플 장비에서 동일한 이벤트가 반복해서 발생했는지 로그를 확인했는가?

[ ] 로그 내에 Ticket opened 또는 Ticket escalated 표현이 포함되어 있는지 확인했는가?

[2단계: 내용 정리 및 판단]

[ ] 원인을 성급하게 단정 짓지 않고, 로그에 나온 관찰 현상만 객관적으로 정리했는가?

[ ] 혼자 판단하기 모호하거나 추가 확인이 필요한 부분을 'Escalation 필요 여부'로 분류하여 표시했는가?

[ ] 정리한 내용을 Handover Memo 또는 Incident Report 초안에 쓰기 좋게 짧고 명확하게 요약했는가?

[3단계: 가상 환경 보안 및 주의사항 검증]

[ ] 실제 운영 장비나 AWS 환경에 접속하지 않고 샘플 데이터만 보았는가?

[ ] ping이나 traceroute 같은 실제 네트워크 진단 명령을 실행하지 않았는가?

[ ] 실제 IP 주소, 실제 티켓 번호, 실제 장비 시리얼 등 실제 정보가 포함되지 않았는가?

[ ] AI 답변 초안에 원인을 단정 짓는 표현이 있어 사람(학생 본인)이 직접 수정했는가?
